Nesse exemplo SEM o computed sempre que atualizarmos o contador 2 será chamado o método pegarResultado().

O que ñ é bom acontecer, já que ele faz um processamento que é desnecessário.

Ver console

{{ contador }} | {{ contador2 }}

{{ pegarResultado() }}

Aqui no 2º exemplo, COM o computed, o método pegarResultado() ñ é chamado de maneira desnecessário.

Isso porque o Vue entende que dentro desse método computado, o valor que foi alterado [contador2] ñ gera nenhum impacto na unidade computada [a unidade computada é o contador 1]

Nesse caso pegarResultado ñ leva parênteses.

Ver console

{{ contador }} | {{ contador2 }}

{{ pegarResultado }}
